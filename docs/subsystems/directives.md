# Directives — FleetRules and the render pipeline

CLAUDE.md, FIX-LOG.md, every per-domain doc that begins with `<!-- AUTO-GENERATED -->`, and every agent's `--append-system-prompt` content all derive from a single source: rows in the `FleetRules` table. The on-disk files are renderings, not authoritative — editing them by hand is what Pattern P18 catches at CI time. This page is the operator reference for the rule slice, the render targets, and the renderer's failure modes.

## Overview

D3 Phase 1 introduced `FleetRules` as the DB-shaped substrate for what previously lived hand-written in CLAUDE.md / SENATE.md / BoS rule files / ISB finder configs. Every row carries a `render_to` slot that dispatches the rendered output:

- `claude-md-file` — the universal-load file (`CLAUDE.md` at repo root). 10 KB target, 20 KB hard cap (`ClaudeMdHardCapBytes`). Rules here apply simultaneously to the operator, to Claude Code building Force, and to every review agent that auto-loads CLAUDE.md from daemon CWD. Almost nothing meets the bar; each row carries a `Justification` field in `bootstrapAudit` explaining the universal-load fit.
- `agent-prompt` — per-agent injection only. `agent_scope` filters to a single agent or comma-separated list (`captain,council,...`). Never written to a shared file; consumed at agent spawn via `AppendFleetRulesToPrompt` / `InjectFleetRulesAgentPrompt` and concatenated under the `SourceTagFleetRules` slot.
- `fix-log` — `FIX-LOG.md` historical-narrative entries; append-only, no size cap (operator-browsed, not auto-loaded).
- `pattern-test-docstring` — lives in the Pattern test file's docstring; CLAUDE.md gets a one-line cross-ref.
- `per-domain-doc:<file>` — domain-specific markdown under `docs/<file>` (e.g. `per-domain-doc:dashboard-conventions.md`, `per-domain-doc:pr-flow-invariants.md`, `per-domain-doc:self-healing.md`).
- `discard` — row kept for history but renders nowhere.

The audit slice in `internal/store/fleet_rules_audit.go` is the seed. `BootstrapFleetRules` is convergent: a re-run against a clean DB inserts; against a drifted DB it refreshes any `created_by='bootstrap'` row whose `content_hash` no longer matches the audit (operator-direct-write rows are never touched).

## Components

- `internal/store/fleet_rules_audit.go` — the audit slice (`bootstrapAudit []FleetRuleSeed`). Each seed carries `RuleKey`, `Section` (originating CLAUDE.md H2), `Category`, `AgentScope`, `RenderTo`, `EnforcedBy` (a Pattern test ID or `trust-only`), `Justification` (documentation-only — explains the `RenderTo` choice, especially for `claude-md-file` rows), `Content`. Per-narrative `FIX-LOG.md` fragments are embedded from `internal/store/fixlog/*.md` (the same text that previously lived hand-written in `FIX-LOG.md`) so backticks, code spans, and Pattern P11 cheat-shape literals stay verbatim without Go-source escaping.
- `internal/store/fleet_rules_bootstrap.go` — `BootstrapFleetRules`, `assertAllSectionsCovered` (refuses to drop a CLAUDE.md H2 by accident), `FleetRuleSeed`, `setRuleVersion`, `contentHash`. Convergent on `content_hash` mismatch.
- `internal/agents/rule_renderer.go` — render dispatchers, one per target: `RenderClaudeMdFile`, `RenderFixLog`, `RenderPerDomainDocs`, `AssemblePerAgentPrompt`. Disk-write helpers: `WriteRenderedClaudeMd`, `WriteRenderedFixLog`, `WriteRenderedPerDomainDocs`. Drift detector: `CheckRenderDrift`. Header constant: `renderHeader = "<!-- AUTO-GENERATED by force render-rules ... -->"`.
- `internal/agents/fleet_rules_inject.go` — `InjectFleetRulesAgentPrompt` (PromptBuilder shape) and `AppendFleetRulesToPrompt` (string shape). Both fail-open: a missing `FleetRules` table or query failure logs but does not interrupt agent startup. Appended bytes are tagged under `SourceTagFleetRules` for `PromptByteAttribution`. Call-site contract: invoke AFTER the static system prompt has been seeded but BEFORE `Build()`.
- `cmd/force/render_rules.go` — `cmdRenderRules` implements `force render-rules [--check] [--use-runtime-db]`. Default opens a fresh `:memory:` DB, bootstraps the audit, renders to disk; `--check` renders to memory and exits 1 on any disk disagreement. `--skip-fix-log` skips FIX-LOG.md on a single invocation. The fresh-DB default is the same code path `TestPattern_P18_RenderCoherence` uses, so CLI output is deterministic regardless of operator-side persistent-DB drift.
- `scripts/pre-commit/render-coherence-check.sh` + Makefile target `render-rules-check` — pre-commit + CI gate.

The bootstrap audit also feeds `EmitCandidate` through the Librarian: hypothesis-emit dogs and operator-direct candidate proposals all land in `PromotionProposals` with `kind='candidate'` and `authored_by='librarian'` (the convention that doubles `authored_by` as the origin column without a schema migration). Engineering Corps's `ListPendingCandidates` is the matching read surface.

## Invariants

1. **All-sections-covered.** `assertAllSectionsCovered` requires every CLAUDE.md H2 heading to appear in at least one seed's `Section` field. Adding a section without a seed is a bootstrap error, not a silent drop.
2. **One active row per rule.** Partial UNIQUE index `idx_fleet_rules_active_key ON FleetRules(rule_key) WHERE active_until = ''` enforces at most one active version per `rule_key`; the lineage index `idx_fleet_rules_key_version (rule_key, version)` keeps the version chain.
3. **Convergent bootstrap.** Re-running `BootstrapFleetRules` against a drifted DB UPDATES `created_by='bootstrap'` rows whose `content_hash` no longer matches the audit. Operator-direct-write rows (`created_by != 'bootstrap'`) are NEVER touched.
4. **CLAUDE.md hard cap.** `RenderClaudeMdFile` refuses to write past `ClaudeMdHardCapBytes` (20 KB). Pattern P17 (`docs/patterns/p17-claude-md-size.md`) catches this in CI; the pre-commit `claude-md-size-check.sh` catches it locally.
5. **Render coherence.** Pattern P18 (`internal/audittools/audit_pattern_p18_render_coherence_test.go`) bootstraps a `:memory:` DB, renders all targets, and byte-compares against on-disk files. Drift means an operator edited `fleet_rules_audit.go` (or its embedded fix-log fragment) without re-running `make render-rules` and re-staging.
6. **`enforced_by` is a real Pattern test or `trust-only`.** Audit-time discipline: every rule names its enforcement seam.
7. **Agent-prompt injection is fail-open.** A missing FleetRules table or query error must not stop an agent from running — the legacy const-based system prompt remains in place; the FleetRules append is purely additive.
8. **CLI default uses fresh DB.** `force render-rules` defaults to a `:memory:` DB so output is deterministic regardless of operator-side persistent-DB drift; `--use-runtime-db` is the explicit opt-in.

## Failure modes

When the renderer trips, the operator surface differs by layer:

- `BootstrapFleetRules` returns an error and the daemon refuses to start when an audit seed references a CLAUDE.md H2 that no longer exists OR when an H2 in CLAUDE.md is not covered by any seed's `Section` field. The fix is to update the audit slice to match the current CLAUDE.md.
- `RenderClaudeMdFile` returns `[RULE-RENDERER OVERFLOW] rendered CLAUDE.md is N bytes; hard cap is M` when content exceeds 20 KB. The fix is either trimming a rule's `Content` or moving it to a `per-domain-doc:<file>` target.
- `make render-rules-check` exits 1 with a list of drifted paths. The fix is `make render-rules` followed by re-staging.
- The `claude-md-drift-watch` dog emits `PromotionProposals` rows; operator ratifies via the EC tab on the dashboard.

## Configuration

`schema/schema.sql` — `FleetRules` columns:

- `id`, `rule_key`, `category` (`architecture`|`schema`|`security`|…), `agent_scope` (`all` or comma-separated agent list), `render_to` (the controlled enum above), `enforced_by` (Pattern test ID or `trust-only`), `content`, `content_hash`, `version`, `active_from`, `active_until` (`''` = active; non-empty = retired), `promoted_by_experiment_id`, `created_by` (`bootstrap` for audit-managed rows; operator email for direct writes), `created_at`.

Renderer-side constants (`internal/agents/rule_renderer.go`):

- `ClaudeMdHardCapBytes = 20 * 1024` — hard cap on rendered CLAUDE.md.
- `renderHeader` — `<!-- AUTO-GENERATED by force render-rules from FleetRules table. ... -->` prepended to every auto-generated file.

Auto-rendered output paths (per `autoRenderedDocPaths` in `internal/audittools/audit_pattern_p_docs_architecture_test.go`):

- `CLAUDE.md` (root) — universal-load.
- `FIX-LOG.md` (root) — historical narrative.
- `docs/dashboard-conventions.md` — dashboard conventions.
- `docs/pr-flow-invariants.md` — PR-flow invariants.
- `docs/self-healing.md` — self-healing rules.

Hand-edits to any of those files are rejected by Pattern P18 + the pre-commit hook; the workflow is "edit `fleet_rules_audit.go`, run `make render-rules`, stage the rendered output."

## Render targets in detail

Each render target has its own dispatcher in `internal/agents/rule_renderer.go`:

| Target | Dispatcher | On-disk path | Cap |
|---|---|---|---|
| `claude-md-file` | `RenderClaudeMdFile` | `CLAUDE.md` | 20 KB hard (`ClaudeMdHardCapBytes`); 10 KB target |
| `fix-log` | `RenderFixLog` | `FIX-LOG.md` | none (operator-browsed) |
| `per-domain-doc:<file>` | `RenderPerDomainDocs` | `docs/<file>` | none |
| `agent-prompt` | `AssemblePerAgentPrompt` | (in-memory; concatenated into the agent's system prompt) | per-agent budget; tracked under `SourceTagFleetRules` in `PromptByteAttribution` |
| `pattern-test-docstring` | (out-of-band; lives in the pattern test file) | `internal/audittools/<test>.go` | none |
| `discard` | (no dispatcher) | (none) | n/a |

Each rendered file begins with the `renderHeader`:

```
<!-- AUTO-GENERATED by force render-rules from FleetRules table.
     Do not edit by hand. Edit the FleetRules row, then run `make render-rules`. -->
```

`RenderClaudeMdFile` orders rules by `(category, rule_key)` so the rendered output is deterministic. `RenderFixLog` likewise; the embedded `internal/store/fixlog/*.md` fragments are the verbatim narrative bodies (preserves backticks and code spans without Go-source escaping).

## Operator surface

```bash
force render-rules                 # bootstrap + render to disk (no-op when nothing changed)
force render-rules --check         # exit 1 if any on-disk file disagrees with the audit-slice render
force render-rules --skip-fix-log  # skip FIX-LOG.md on this invocation
force render-rules --use-runtime-db # use the persistent holocron.db (operator-direct-write rules visible)
make render-rules                  # Makefile alias
make render-rules-check            # drift gate (pre-commit + CI)
```

To add a new universal-load rule:

1. Insert a `FleetRuleSeed` row in `internal/store/fleet_rules_audit.go` with `RenderTo: 'claude-md-file'` AND a non-empty `Justification` explaining the universal-load fit.
2. Run `make render-rules` to regenerate `CLAUDE.md` (and any other targets the rule changed).
3. Stage the audit-slice change AND the rendered file in the same commit. Pattern P18 will fail CI otherwise.

To retire a rule: bump the active row's `active_until = datetime('now')` and let the convergent bootstrap pick up the next version. Operator-direct-write rules survive the bootstrap pass; bootstrap-managed rules refresh from the audit.

The `claude-md-drift-watch` dog (weekly cooldown; see [`dogs.md`](dogs.md)) compares CLAUDE.md invariants against the FleetRules audit and emits drift candidates as `PromotionProposals` rows for operator ratification. This is the steady-state catch when an operator edits CLAUDE.md by hand and forgets to mirror the change into the audit slice — Pattern P18 catches it at commit time, the dog catches it weekly for any drift that slipped past local hooks.
