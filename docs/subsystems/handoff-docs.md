# Synthetic handoff documentation

D10 ships two paired surfaces that auto-write reviewer-facing prose so a human (or an Astromech) can read a draft PR or browse a repo without reverse-engineering the convoy and the architecture from scratch:

1. **`PRHandoffSynthesis`** — a Diplomat task type that posts a "how to read this PR" comment on every draft PR opened by an enabled repo's convoy.
2. **`dogArchitectureDocRender`** — a 1-hour-cadence dog that re-renders `<repo>/ARCHITECTURE.md` for every repo with the opt-in flag set.

Both are gated on `Repositories.handoff_synthesis_enabled` (default 0). Until an operator flips the flag for a specific repo, neither surface fires for that repo. The validation experiment (`experiments/D10-handoff-synthesis.yaml`) gates expansion: keep the feature only if the auto-narrative measurably shortens human review cycles.

## Overview

Reviewer-narrative comment (`PRHandoffSynthesis`):

- Diplomat composes a 4-section Markdown comment ("What changed", "Where to look first", "What the fleet's review gates said", "Things you might want to push back on") using the existing Diplomat capability profile and a new `PromptVersion=diplomat-pr-handoff-v1`.
- The narrative draws from convoy diff (file list), most recent ConvoyReview verdict + outcomes JSON, Council / Captain rulings (extracted from `TaskHistory`), and any Senate reviews on the convoy's Feature ancestor.
- An auto-generated footer is appended after the model returns so reviewers always know the comment came from the fleet (the system prompt forbids the model from doing this itself).

Architecture-doc render (`dogArchitectureDocRender`):

- Walks every repo with `handoff_synthesis_enabled=1`, resolves the digest via `librarian.Client.BuildArchitectureDoc`, and writes `<repo>/ARCHITECTURE.md`.
- 1-hour cadence with a no-op fast path on unchanged main SHA. Functionally equivalent to "on every merge to main" from the operator's perspective (worst-case 1h staleness); avoids wiring a new git-hook fan-out path.
- The render reuses D6's `BuildRepoDigest` so the digest-assembly seam stays collapsed across onboarding, Senate bootstrap, and architecture-handoff. The `ArchitectureDoc` data shape is pure-data; the dog (not the librarian) writes the file.

The two surfaces are independent at run time but share enrolment and the AUTO-GENERATED-only invariant (pre-commit hook rejects hand-edits).

## Components

- **`internal/agents/diplomat_pr_handoff.go`** — `QueuePRHandoffSynthesis(db, convoyID, experimentArm)` enforces the per-repo opt-in flag at queue time AND `runPRHandoffSynthesis` re-checks at run time so a flag flip mid-flight gracefully no-ops. `prHandoffSynthesisSystemPrompt` anchors the reviewer-narrative shape; `appendHandoffSynthesisFooter` stamps the AUTO-GENERATED footer after the LLM call. Per-repo, per-(convoy, ask-branch) processing — a multi-repo convoy can have one repo enrolled and another not.
- **`internal/agents/diplomat.go`** — Diplomat dispatcher claims `PRHandoffSynthesis` bounties (alongside `DraftPROpen`, `PRReviewTriage`, `ConvoyReview`, etc.) and routes them to `runPRHandoffSynthesis`.
- **`internal/agents/dog_architecture_render.go`** — `dogArchitectureDocRender` walks enabled repos, calls `librarian.Client.BuildArchitectureDoc`, and writes `<repo>/ARCHITECTURE.md`. The `architectureDocAutoGeneratedHeaderPrefix` (`<!-- AUTO-GENERATED by \`dogArchitectureDocRender\``) is stamped by the librarian renderer; the dog refuses to write a body that does not start with the prefix (renderer regression guard).
- **`internal/agents/dogs.go`** — registers `architecture-doc-render` in `dogCooldowns` (1-hour cadence), `dogOrder`, and `runDog` dispatch. The cadence comment explains the safety-net role: a missed merge-event doesn't leave the doc stale longer than an hour.
- **`internal/clients/librarian/inprocess_d10.go`** — `BuildArchitectureDoc` implementation. Reuses `BuildRepoDigest` and renders the architecture-level Markdown body (`## Subsystem map`, `## Public interfaces`, `## Recent activity`, `## Deployment + data shape`, `## Pointers`). The "Pointers" section is a 1-line `see <file>` reference for each conventions file — never an inlined CLAUDE.md excerpt.
- **`internal/store/tasks.go`** — `PRHandoffSynthesis` is registered in `InfrastructureTaskTypes` so its bounties are classified correctly in dashboards / cost reports / cleanup sweeps.
- **`scripts/pre-commit/architecture-md-check.sh`** — pre-commit hook that rejects a staged `ARCHITECTURE.md` whose first staged line does not start with the AUTO-GENERATED prefix. Reads the staged blob (not the working tree); mirrors the D6 ONBOARDING.md hook shape.
- **`internal/clients/librarian/inprocess_d10_test.go`** — `TestArchitectureMdNotDuplicateOfClaudeMd` is the anti-cheat enforcement: a sliding-window detector (window=81) plants distinctive 120-char lines into a synthetic CLAUDE.md and asserts no contiguous >80-char run from CLAUDE.md appears in the rendered ARCHITECTURE.md.
- **`experiments/D10-handoff-synthesis.yaml`** — paired-run experiment scaffold. Two arms (`control_off`, `treatment_on`) at the per-repo level; primary metric `time_to_review_close`, secondary `review_comment_count`; T+30 operator verdict (keep / drop).

## Invariants

1. **Default OFF, every layer.** `Repositories.handoff_synthesis_enabled INTEGER NOT NULL DEFAULT 0` (anti-cheat #A made structural). `QueuePRHandoffSynthesis` returns `(0, nil)` if no repo in the convoy is enabled. `runPRHandoffSynthesis` re-checks at run time and completes as a no-op if the flag flipped off in flight. `dogArchitectureDocRender` walks only repos with the flag set.
2. **AUTO-GENERATED-only artefacts.** `ARCHITECTURE.md` carries the `<!-- AUTO-GENERATED by \`dogArchitectureDocRender\`` prefix as its first line; the pre-commit hook rejects any staged `ARCHITECTURE.md` missing the prefix. The `PRHandoffSynthesis` comment carries the `_AUTO-GENERATED reviewer narrative for convoy ..._` footer (appended post-LLM by `appendHandoffSynthesisFooter`).
3. **No `ARCHITECTURE.md` duplicating `CLAUDE.md`.** `TestArchitectureMdNotDuplicateOfClaudeMd` is a sliding-window detector at window=81. The "Pointers" section emits 1-line references only — never inline excerpts. CLAUDE.md is the invariants doc; ARCHITECTURE.md is the system map. The two stay distinct by construction.
4. **Idempotency at queue time.** `QueuePRHandoffSynthesis` uses the idempotency key `pr-handoff-synthesis:<convoyID>` so a re-queue while one is already pending is a no-op (mirrors `QueueConvoyReview`'s contract).
5. **Convoy-scoped via `convoy_id`.** The bounty row carries `convoy_id` populated at queue time so the convoy-scoped index (`idx_bounty_convoy_status`) catches the row — never `payload LIKE '%convoy_id%'`. New infrastructure task types must follow this rule per the schema-conventions invariant.
6. **No silent failures.** Every error path in `runPRHandoffSynthesis` terminates in `store.FailBounty` or `store.UpdateBountyStatus`. A partial multi-repo run records `posted=...` / `skipped=...` / `failures=...` in the failure message so the operator can diagnose. Posted-comment-but-missing-audit-row is preferred over the reverse — the comment is on GitHub even if `InsertPRHandoffSynthesis` failed.

## Configuration

Per-repo opt-in:

```sql
UPDATE Repositories
   SET handoff_synthesis_enabled = 1
 WHERE name = '<repo>';
```

Or via the experiment scaffold's enrolment step (`force experiment author experiments/D10-handoff-synthesis.yaml`). The schema column is `Repositories.handoff_synthesis_enabled INTEGER NOT NULL DEFAULT 0` — additive ALTER lands clean on existing DBs (no backfill; existing repos remain off until the operator opts in).

Audit table (one row per posted reviewer-narrative comment):

```sql
CREATE TABLE PRHandoffSyntheses (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    convoy_id      INTEGER NOT NULL REFERENCES Convoys(id),
    pr_url         TEXT NOT NULL,
    posted_at      TEXT NOT NULL,
    experiment_arm TEXT NOT NULL DEFAULT '',
    comment_id     INTEGER NOT NULL DEFAULT 0   -- gh pr comment doesn't surface the REST id; v2 swaps to gh api
);
CREATE INDEX idx_pr_handoff_convoy ON PRHandoffSyntheses(convoy_id);
```

Cooldown (1h) for the architecture-doc dog is registered in `dogCooldowns`. Operators who want to force a re-render before the cooldown elapses can run `force dogs run architecture-doc-render` directly; the dog's no-op fast path keeps the cost trivial when nothing changed.

## Operator surface

Enrol a repo:

```bash
force repos set-handoff-synthesis <repo> on    # flips Repositories.handoff_synthesis_enabled to 1
force repos set-handoff-synthesis <repo> off   # back to default
```

Trigger a one-shot architecture render (otherwise the dog catches up within the hour):

```bash
force dogs run architecture-doc-render
```

Inspect audit rows:

```bash
sqlite3 holocron.db "SELECT convoy_id, pr_url, posted_at, experiment_arm
                       FROM PRHandoffSyntheses
                      ORDER BY id DESC LIMIT 20;"
```

Drive the validation experiment:

```bash
force experiment author experiments/D10-handoff-synthesis.yaml   # arm assignment
# ... 30 days of PR throughput ...
force experiment verdict D10-handoff                             # T+30 keep / drop
```

When an Astromech opens a PR that touches `ARCHITECTURE.md`, the pre-commit hook (`scripts/pre-commit/architecture-md-check.sh`) rejects the commit unless the staged file's first line carries the AUTO-GENERATED prefix. The hook's error message names the offending paths and points the operator at the dog (`force dogs run architecture-doc-render`).

The orchestrator's own `ARCHITECTURE.md` is `.gitignore`'d at the root for the same reason `ONBOARDING.md` is — a smoke-run of the dog must not pollute `git status`. Operators who want a renderer-produced sample committed must rename it.
