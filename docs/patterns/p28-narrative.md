# Pattern P28 — Narrative is generated

## Rationale

NarrativeRenders rows are produced by exactly one production code
path — the renderer in `narrative_renderer.go`. Two enforcement points
matter:

1. **Single insert site.** Only one production file may issue
   `INSERT INTO NarrativeRenders`. Multiple writers would mean
   competing prose styles and untraceable provenance.
2. **Prompt-in-code.** The narrative prompt template lives in
   `internal/agents/narrative_prompts/v1.go`, not in a SystemConfig
   row. Code-stored prompts version with the binary; DB-stored
   prompts drift.

Originates in D3 P6A.7.

## What it checks

`TestPattern_P28_NarrativeIsGenerated`:

1. Greps `internal/agents`, `internal/dashboard`, `internal/store`
   for `INSERT INTO NarrativeRenders` (production only). Asserts
   exactly one match AND that the match's file ends in
   `narrative_renderer.go`.
2. Asserts `internal/agents/narrative_prompts/` exists.
3. Asserts `narrative_prompts/v1.go` declares both `PromptVersion`
   and `PromptTemplate` constants.

## How it fails

```
Pattern P28 violation: NarrativeRenders inserted from 2 files; want exactly 1 (narrative_renderer.go).
Files: internal/agents/narrative_renderer.go, internal/agents/captain.go
```

Or for the prompt-in-code prong:

```
Pattern P28: PromptVersion const missing from narrative_prompts/v1.go
```

## How to fix

If a second writer appears, fold its narrative emission into
`narrative_renderer.go` (or call into it). For prompt drift, ensure
the constants stay defined in `narrative_prompts/v1.go` — DO NOT move
the prompt body into a SystemConfig row.
