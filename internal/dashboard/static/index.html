Fleet Command Center

Pulse Briefing Reflection Legacy

Pulse

Ambient fleet view. Live narrative panel + fleet vital signs land in 6A.7 / 6A.8.

Briefing

Decision queue + conversational triage. Conversational briefings land in 6A.10.

Drill — search across convoys / tasks / events

Free-text search the fleet's transcript / git-op / task history. Tap a result to drill into the convoy or task timeline.

Drill — by convoy

Enter a convoy id to load its unified event stream (transitions, mail, git ops, LLM transcripts).

Drill — by task

Enter a task id to load its event stream (single-task drill).

Drill — single event detail

Inspect the full body of one event (transcript, git op, mail).

Replay — re-run a historical decision

Re-run a past Captain / Council / Medic / Convoy-review decision under the current prompt version. Pure read on live state — only ReplayResults + replay's own LLMCallTranscripts row are written.

Annotations — operator notes on events

Browse annotations the operator has flagged on events. CRUD via `force annotate` CLI (write paths are CLI-only; this view is read-only).

Calibration scoreboard

Confirmed-vs-overridden audit decisions per stakes bucket. Buckets surface drift early — accuracy_pct dropping in one bucket while staying steady in another is the signal.

Fleet learning panel

Weekly auto-rendered summary of how the fleet itself is changing — promotions ratified, features filed, spec amendments, convoy-review activity.

5-min retro generator

Generate a markdown retro for the trailing 7 days — top win, top frustration, suggested experiment. Save to docs/retros/<date>.md (operator commits manually).

Security findings

      
List of every BoS / ISB finding. Filter by bureau / disposition / rule. Operators can resolve a finding (close, suppress, or override with a bypass audit id).

      

        
        
        
        
        
      

      

    


    

      
Per-rule precision metrics

      
Per-rule firing volume, true-positive / false-positive split, derived precision, and anti-cheat ramp status (advise rules graduate to block after 30 clean firings).

      

        
        
      

      

    


    

      
Override audit

      
Every // BOS-BYPASS / // ISB-BYPASS comment that downgraded a finding. Filter by bureau, rule, or audit id. The full audit trail of operator-acked bypasses lives here.

      

        
        
        
        
        
      

      

    

  


  
  

    

      
Architecture Health

      
Monthly longitudinal architectural-debt report — every BoS finding aggregated by (rule, repo, author_type). Generated by the architecture-health-report dog (1st of each month, 00:00 UTC). The ⚠️ flag means astromech violation_count exceeds human violation_count for that rule.

      

        
        
        
      

      

      

    

  


  
  

    

      
      
    


    

      
Senate chambers

      
Active Senators consulted by the Chancellor on each Feature. Each Senator has a repo / team scope and an onboarding lifecycle state.

      

        
        
      

      

    


    

      
Review log

      
Every Senator's verdict on each Feature. Filter by feature_id, senator, or position. Click an id to see the cited memories that shaped the verdict.

      

        
        
        
        
      

      

      

    

  


  
  

    

      
Notifications

      
Mode-of-the-day for operator-facing alerts. The active preset routes most categories; per-category overrides + DND layer on top. Tier-1 categories require a confirmation modal before silencing — the carve-out exists to prevent the fleet from going silent on a real escalation.

Set Do Not Disturb

        
DND silences every category except spend_cap_e_stop and consumer_breakage. Server-side cap: 14 days from now.

        

          
          
          
          

          

            
            
          

        

      

    


    
    

      

        
Tier-1 silencing — confirm

        
Tier-1 categories signal the fleet is blocked. Silencing them means a real escalation will be invisible until you actively check the dashboard.

        

        

          
          
        

      

    


    
    

      

        
Save current overrides as a preset

        
Captures the current per-category overrides as a YAML preset block. The block is written to /tmp/notifications-preset-<name>-<ts>.yaml.diff; paste it into config/notifications.yaml via a normal PR.

        

          
          
          

          

            
            
          

        

      

    

  


  
  

    

      
Save current filter

      
Captures the current filter + sort as a named query you can recall in one click. Stored as source=dashboard; export to YAML and commit to config/dashboard.yaml to make it canonical.
